# CookieMusic — 毕业设计功能总结报告

## 项目概述

CookieMusic 是一个基于 AI 的音乐创作与分享平台。用户通过自然语言描述（提示词）控制 AI 生成音乐作品，支持歌曲（带人声）和纯音乐（器乐）两种模式。平台还包含用户系统、积分充值、推荐算法、音乐社交（点赞/播放/分享）等完整功能。

**项目地址**: https://github.com/hugotee/musical

## 五、AI 模型方案

### 5.1 歌曲（带人声）→ 天谱乐 TemPolor v4.5

- **调用方式**: HTTP POST → `https://api.tianpuyue.cn/open-apis/v1/song/generate`
- **生成模式**: 异步（提交任务 → 立即返回 → 回调通知 / 轮询）
- **生成时长**: 最长 270 秒（第三方接口能力，当前前端默认使用 V3）
- **支持功能**: 人声歌唱、歌词、曲风/情绪控制
- **积分消耗**: 当前前端默认 V3 模型 30 积分/首；V3.5 为历史/扩展配置

### 5.2 纯音乐（器乐）→ Meta MusicGen Small

- **模型**: facebook/musicgen-small（300M 参数）
- **论文**: Copet et al., "Simple and Controllable Music Generation", 2023
- **部署**: 本地 Flask 服务 (port 8092)
- **硬件加速**: Apple Silicon MPS + float16 半精度（内存 ~2GB）
- **生成模式**: 同步（等待生成完成，约 90 秒 / 30 秒音频）
- **提示词优化**: 中文曲风/情绪自动翻译为英文音乐术语（18 种曲风 + 15 种情绪映射）
- **积分消耗**: 当前前端默认 V3 模型 25 积分/首；数据库中保留 V3.5 扩展配置

## 六、推荐算法

实现了完整的混合推荐系统（`RecommendationServiceImpl`），799 行代码，支持 6 种策略：

### 6.1 基于内容的推荐 (Content-Based)
- 构建用户画像：统计用户点赞音乐的曲风（musicGener）、情绪（musicEmotion）、人声（musicSex）偏好频次
- 候选音乐与用户画像计算余弦相似度
- 冷启动阈值：点赞 < 3 首时降级为热度排序

### 6.2 User-Based 协同过滤
- 加载全量点赞记录构建 `user→likedMusicIds` 映射
- 计算用户间余弦相似度：`|A ∩ B| / sqrt(|A| × |B|)`
- 选取 Top 10 最相似用户，聚合他们喜欢但目标用户未听过的音乐

### 6.3 Item-Based 协同过滤
- 构建 `music→likingUserIds` 反向映射
- 对每首候选音乐，计算与用户已点赞音乐的用户集合相似度
- 取最大相似度作为 Item-CF 分数

### 6.4 热度排序 (Popularity)
- 综合播放量（权重 0.5）和点赞数（权重 2.0）
- 时间衰减：`exp(-days / 30)`，半衰期 30 天

### 6.5 混合推荐 (Hybrid，默认)
- 内容 0.25 + UserCF 0.25 + ItemCF 0.25 + 热度 0.15 + 时间衰减 0.10

## 七、已知问题与改进方向

### 安全问题（适合作为"未来工作"写入论文）
1. **后台认证机制较简单** — 管理端使用固定管理员账号 + Redis token，生产环境需接入更完整的权限模型
2. **密码 MD5 无盐存储** — 应迁移至 BCrypt/Argon2
3. **配置文件明文存储密钥** — API Key、数据库密码提交到 Git
4. **MyBatis order by 使用 ${}** — 存在 SQL 注入理论风险
5. **任意文件读取** — `/file/getResource` 路径遍历防护不足
6. **-Ddev 模式绕过所有认证** — 生产环境需移除

### 功能改进
1. 音乐时长可选（10s/15s/30s）
2. MusicInfo.lyrics 存储天谱乐返回的结构化歌词
3. WAV 转 MP3 真实转码（当前仅改后缀，Safari 兼容性差）
4. 付款码有效期目前由后端固定为 30 分钟，可改为后台可配置
5. 音乐封面可由 AI 自动生成
